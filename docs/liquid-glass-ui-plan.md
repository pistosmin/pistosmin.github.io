# 리퀴드 글래스 UI 정리 계획

## 목적

이 페이지의 최우선 목표는 "오래 읽히는 개인 블로그"다. 리퀴드 글래스는 사이트의 분위기를 만드는 보조 재료로 쓰되, 글을 읽는 행위보다 앞에 나오면 안 된다.

따라서 수정 방향은 효과를 더 많이 바르는 것이 아니라, 유리 표면이 필요한 곳과 필요하지 않은 곳을 분리하는 것이다. 탐색, 검색, 목차, CTA처럼 사용자가 움직이는 부분에는 굴절과 광택을 주고, 본문과 긴 설명문은 종이처럼 안정적인 표면 위에 둔다.

## 현재 상태 요약

- `src/components/Refractive.tsx`에 `bar`, `pill`, `icon`, `chip`, `panel` 프리셋이 이미 있다. 이 파일을 리퀴드 글래스의 단일 진입점으로 유지하는 것이 좋다.
- `src/styles/global.css`에는 색상, 표면, 그림자, 글래스 관련 CSS 변수가 이미 정의되어 있다. 새 디자인도 이 토큰 계층을 확장해서 가야 한다.
- 홈은 `src/pages/index.astro`에서 히어로, 프로필 카드, 카테고리 인덱스, 최근 글, 노트, 프로젝트로 구성되어 있다.
- 글 상세는 `src/pages/posts/[...slug].astro`에서 본문과 목차를 2열로 배치한다. 본문 자체는 비교적 차분하지만 목차, 검색, 관련 글 카드에 표면 위계를 더 줄 수 있다.
- 현재 모바일 캡처 기준으로 히어로 설명 문장이 오른쪽으로 잘리는 문제가 보인다. 우선 수평 오버플로우부터 잡아야 한다.
- 데스크톱 캡처 기준으로 첫 화면의 큰 제목과 우측 글래스 카드들은 분위기는 있으나, 글을 읽는 블로그라는 신호보다 시각 실험의 비중이 크게 느껴진다.

## 디자인 방향

### 비주얼 명제

따뜻한 종이 배경 위에 얇은 투명 소재가 얹힌 편집형 블로그로 정리한다. 텍스트는 선명하고 정적이어야 하며, 유리 효과는 사용자의 시선을 안내하는 짧은 순간에만 반응한다.

### 원칙

1. 본문은 유리 뒤에 두지 않는다.
2. 글래스 표면은 탐색, 액션, 보조 정보에만 쓴다.
3. 한 화면에서 강한 굴절 표면은 2-3개 이하로 제한한다.
4. 카드보다 레이아웃을 먼저 정리한다. 반복 아이템이 아닌 섹션 전체를 카드처럼 감싸지 않는다.
5. 모바일에서 모든 텍스트는 수평 스크롤 없이 읽혀야 한다.
6. 다크 모드에서도 배경과 글래스가 한 덩어리로 뭉개지지 않게 대비 단계를 분리한다.

## 정보 구조

### 홈

홈 첫 화면은 "누가 쓰는 어떤 블로그인가"를 바로 보여줘야 한다.

- 히어로 텍스트는 3-4줄 안에서 끝내고, 모바일에서는 제목과 설명문 행 길이를 더 짧게 제한한다.
- CTA는 히어로 설명 바로 아래에 확실히 보이게 둔다. `글 읽기`를 1순위, `작업물 보기`를 2순위로 둔다.
- 우측 프로필/카테고리 영역은 "유리 위젯 묶음"으로 유지하되, 본문보다 화려해 보이지 않도록 대비와 그림자를 낮춘다.
- 카테고리 인덱스는 현재 2열 그리드 대신 데스크톱에서는 compact grid, 모바일에서는 단일열 또는 가로 스크롤 칩 중 하나로 정리한다. 모바일 단일열이 가장 안전하다.
- 최근 글 영역은 첫 화면 아래에 살짝 보이도록 유지하되, 제목과 카드 사이 여백을 줄여 실제 읽을 콘텐츠가 빨리 등장하게 한다.

### 글 목록

글 목록은 리퀴드 글래스의 실험장이 아니라 탐색 화면이다.

- 카테고리 내비게이션만 `chip` 형태의 리프랙티브 컨트롤로 유지한다.
- 글 카드는 반투명보다 읽기 좋은 불투명 표면을 우선한다.
- 카드 내부의 카테고리, 날짜, 태그는 색을 낮춰 제목과 설명이 먼저 읽히게 한다.

### 글 상세

글 상세는 가장 보수적으로 다룬다.

- 본문 컬럼은 `680-720px` 범위로 유지하고, 행간은 현재처럼 넉넉하게 둔다.
- 목차는 글래스 패널로 둘 수 있지만, 목차 내부 텍스트 배경은 충분히 안정적으로 만든다.
- 본문 중간에는 글래스 장식을 넣지 않는다. 단, 리퀴드 글래스를 설명하는 해당 글에는 별도 데모 컴포넌트를 한 번만 넣을 수 있다.
- 관련 글 카드는 본문을 다 읽은 뒤의 탐색 요소이므로 약한 글래스 또는 일반 카드 모두 가능하다. 우선 일반 카드로 두는 편이 낫다.

## 컴포넌트 정리안

### 기존 컴포넌트 조정

#### `Refractive.tsx`

리퀴드 글래스 효과의 설정을 이 파일에 모은다.

- `bar`: 상단 내비게이션 전용
- `pill`: 주요 CTA와 검색 버튼
- `icon`: 테마 전환, 닫기 버튼
- `chip`: 카테고리, 작은 링크
- `panel`: 검색 다이얼로그, 목차 같은 보조 패널

추가로 고려할 프리셋:

- `subtlePanel`: 목차, 프로필처럼 텍스트가 들어가는 패널용. blur와 specular를 낮춘다.
- `readingControl`: 글 상세에서 쓰는 작은 액션용. 굴절보다 선명도를 우선한다.

#### `Header.tsx`

헤더는 사이트에서 가장 자주 보는 글래스 요소다.

- 데스크톱에서는 지금처럼 떠 있는 바 형태를 유지한다.
- 모바일에서는 높이를 줄이고, 내비게이션 링크가 줄바꿈되며 첫 화면을 과하게 밀어내지 않도록 한다.
- 스크롤 중에는 배경 불투명도를 조금 올려 링크 가독성을 확보한다.

#### `SearchDialog.tsx`

검색은 글래스 효과가 잘 어울리는 기능이다.

- 다이얼로그 외곽은 `panel` 또는 `subtlePanel`로 둔다.
- 입력창과 검색 결과는 불투명 표면에 가깝게 둔다.
- 결과 excerpt는 유리 효과를 피하고, 하이라이트가 잘 보이도록 대비를 높인다.

#### `PostCard.astro`, `ProjectCard.astro`

카드는 콘텐츠의 입구다.

- 카드 전체를 강한 유리로 만들지 않는다.
- hover 상태에서만 아주 약한 테두리 광택이나 translate를 준다.
- featured 카드만 배경 색을 조금 다르게 주고, 굴절 효과는 넣지 않는다.

### 신규 컴포넌트 후보

#### `GlassSurface`

CSS 기반 표면 컴포넌트다. React 리프랙티브가 필요 없는 정적 영역에 쓴다.

사용처:

- 프로필 카드
- 목차 패널
- 검색 다이얼로그 외곽
- 노트 패널

핵심은 "글래스처럼 보이되 텍스트는 안정적으로 읽히는 표면"이다.

#### `ReadingProgress`

글 상세 상단 또는 헤더 아래에 얇은 진행률 바를 둔다.

- 글을 읽는 페이지에서만 표시한다.
- 유리 막대처럼 보이게 할 수 있지만 높이는 낮게 유지한다.
- `prefers-reduced-motion`에서는 애니메이션 없이 동작한다.

#### `ArticleReadingRail`

현재 `toc`를 확장한 컴포넌트다.

- 목차
- 읽기 시간
- 현재 섹션 강조
- 위로 가기 액션

단, 모바일에서는 본문 위에 접힌 목차 또는 단순 링크 목록으로 전환한다.

#### `LiquidGlassDemo`

`liquid-glass-blog-ui.mdx` 글에서만 사용하는 데모 컴포넌트다.

- `기본 표면`, `글래스 컨트롤`, `강한 굴절`을 비교한다.
- 데모 안의 텍스트는 짧게 유지한다.
- 사이트 전체 UI에 설명문을 넣는 대신, 설명이 필요한 내용은 이 글 안에서만 보여준다.

#### `FeaturedWritingShelf`

홈의 최근 글 영역을 더 편집적으로 정리하는 컴포넌트다.

- 첫 번째 글은 큰 텍스트 중심 카드
- 나머지 글은 작고 조용한 리스트
- 카드 그리드의 장식보다 제목, 설명, 날짜가 먼저 보이게 한다.

## 기믹 아이디어

기믹은 "계속 보이는 장식"보다 "상호작용 순간의 반응"으로 제한한다.

- 포인터 위치를 따라가는 미세 하이라이트: 이미 `--liquid-x`, `--liquid-y`가 있으므로 CTA, 칩, 아이콘 버튼에만 적용한다.
- 헤더 압축 상태: 스크롤 후 헤더 배경을 조금 더 단단하게 만들고 그림자를 낮춘다.
- 검색 다이얼로그 입장 모션: 120-160ms 정도의 짧은 scale/fade. 검색 결과는 stagger 없이 즉시 읽히게 한다.
- 목차 현재 섹션 표시: 글래스 배경보다 왼쪽 얇은 accent 라인으로 처리한다.
- 카테고리 칩 active 상태: 굴절을 세게 하기보다 배경 불투명도와 텍스트 대비를 올린다.
- 글 제목 앵커 진입 시 highlight: 해시 링크로 이동했을 때 제목 배경에 짧은 accent tint를 준다.

피해야 할 것:

- 본문 단락 뒤의 blur 표면
- 모든 카드를 리프랙티브 컴포넌트로 감싸기
- 배경 장식만을 위한 큰 빛 덩어리
- 텍스트 위에 움직이는 굴절
- 모바일에서 hover 전제 인터랙션

## CSS 토큰 정리

현재 토큰을 유지하되 표면 역할을 더 명확히 나눈다.

- `--surface-readable`: 본문, 카드, 검색 결과처럼 텍스트가 긴 영역
- `--surface-glass`: 장식성 있는 글래스 패널
- `--surface-glass-strong`: 헤더, 검색 다이얼로그처럼 배경과 분리되어야 하는 영역
- `--line-glass`: 밝은 유리 테두리
- `--line-readable`: 일반 콘텐츠 테두리
- `--shadow-glass`: 떠 있는 유리 표면
- `--shadow-readable`: 카드와 본문 주변의 낮은 그림자

`backdrop-filter`와 리프랙티브 효과는 progressive enhancement로 보고, 지원되지 않아도 레이아웃과 대비가 유지되어야 한다.

## 단계별 작업 계획

### 1단계: 가독성 및 모바일 오버플로우 수정

목표는 아무 효과를 추가하지 않고 읽기 문제를 먼저 없애는 것이다.

- 모바일 히어로 설명문이 잘리는 원인을 확인한다.
- `#content`, `.hero-copy`, `.hero-visual`, `.profile-card`, `.hero-index-link`의 min-width와 overflow를 점검한다.
- 한국어 제목과 설명문에 대해 `word-break`, `overflow-wrap`, max-width를 조정한다.
- 390px, 430px, 768px, 1440px 폭에서 수평 스크롤이 없는지 확인한다.

검증:

- `npm run check`
- `npm run build`
- 모바일/데스크톱 스크린샷 비교

### 2단계: 표면 시스템 정리

목표는 리퀴드 글래스 적용 범위를 한눈에 관리할 수 있게 만드는 것이다.

- `global.css`의 글래스 관련 변수를 역할별 토큰으로 정리한다.
- `Refractive.tsx` 프리셋을 실제 사용처 기준으로 재명명하거나 주석으로 용도를 고정한다.
- 정적 패널에는 CSS 클래스 기반 `glass-surface`를 쓰고, 인터랙티브 컨트롤에만 `RefractiveLink`, `RefractiveButton`, `RefractiveSurface`를 쓴다.

검증:

- 라이트/다크 모드에서 헤더, 검색, 카테고리 칩, 목차 대비 확인
- `prefers-reduced-motion`에서 어색한 움직임이 없는지 확인

### 3단계: 홈 히어로 재정렬

목표는 첫 화면에서 블로그의 성격과 읽기 행동이 더 빨리 보이게 하는 것이다.

- 히어로 제목 크기를 데스크톱과 모바일에서 각각 조정한다.
- 설명문 행 길이를 줄이고 CTA를 더 명확히 노출한다.
- 프로필 카드와 카테고리 인덱스를 `subtlePanel` 계열로 낮춘다.
- 최근 글 섹션의 시작 위치를 더 끌어올려 첫 화면 아래에 콘텐츠 힌트가 보이게 한다.

검증:

- 첫 화면에서 `글 읽기` CTA가 보이는지 확인
- 모바일에서 프로필/카테고리 카드가 화면 폭을 넘지 않는지 확인

### 4단계: 글 목록과 글 상세 정리

목표는 탐색과 읽기 흐름을 안정화하는 것이다.

- 글 목록의 카테고리 nav는 글래스 칩으로 유지한다.
- 글 카드는 일반 표면 중심으로 두고, hover에서만 약한 반응을 준다.
- 글 상세 목차를 `ArticleReadingRail` 또는 개선된 `.toc`로 정리한다.
- 모바일 글 상세에서는 목차를 본문 위 접힘 패널 또는 단순 링크 목록으로 전환한다.

검증:

- 긴 제목, 긴 태그, 긴 설명문이 카드 밖으로 밀리지 않는지 확인
- 글 상세 본문이 1열에서 안정적으로 읽히는지 확인

### 5단계: 선택적 인터랙션 추가

목표는 분위기를 만들되 피로도를 낮게 유지하는 것이다.

- `ReadingProgress`를 글 상세에 추가한다.
- 검색 다이얼로그 입장 모션을 정리한다.
- 목차 현재 섹션 표시를 추가한다.
- `LiquidGlassDemo`를 해당 글에만 추가한다.

검증:

- 키보드 포커스가 모든 컨트롤에서 보이는지 확인
- 모션 축소 환경에서 기능이 유지되는지 확인

## 우선순위

1. 모바일 오버플로우와 텍스트 가독성
2. 글래스 효과 적용 범위 제한
3. 홈 히어로와 CTA 재정렬
4. 글 상세 목차와 읽기 진행 상태
5. 리퀴드 글래스 데모 컴포넌트

## 완료 기준

- 모바일 390px 기준 수평 스크롤이 없다.
- 본문, 카드, 목차, 검색 결과 텍스트가 라이트/다크 모드에서 모두 읽힌다.
- 리프랙티브 컴포넌트는 탐색과 액션 위주로만 쓰인다.
- 글 상세 본문에는 강한 유리 효과가 없다.
- `npm run check`와 `npm run build`가 통과한다.
- 데스크톱과 모바일 스크린샷에서 첫 화면의 목적이 분명하다.

## 권장 작업 순서

먼저 1단계와 2단계를 작은 PR 또는 한 작업 단위로 묶는 것이 좋다. 지금 가장 큰 위험은 멋의 방향이 아니라 읽기 기본기가 흔들리는 것이다.

그 다음 홈 히어로를 손보고, 마지막에 글 상세의 보조 인터랙션과 데모 컴포넌트를 넣는다. 이렇게 하면 다른 세션에서 진행 중인 코드와도 충돌이 덜 나고, 각 단계마다 시각적으로 검증하기 쉽다.
