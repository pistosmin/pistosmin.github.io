리퀴드 글래스 효과는 처음 켰을 때 꽤 그럴듯하다. 버튼 위에 빛이 흐르고, 배경이 살짝 굴절되고, 카드가 떠 있는 것처럼 보이면 사이트가 갑자기 비싸 보인다. 문제는 블로그가 "비싸 보이는 화면"보다 "오래 읽히는 글"에 더 가까운 제품이라는 점이다.

`pistoslog` 초기 버전에서도 이 욕심을 냈다. Refractive runtime, `backdrop-filter`, 큰 blur, fixed background, pointer tracking을 한꺼번에 얹었다. 로컬에서는 분위기가 좋았지만, 브라우저 캡처와 배포본에서 안정성이 흔들렸다. 더 불편했던 것은 원인도 한 번에 보이지 않았다는 점이다. 라이브러리 문제인지, CSS blur 문제인지, React island 구조 문제인지 섞여 있었다.

그래서 한동안 효과를 켰다 끄는 실험을 했다. Refractive만 꺼보고, blur만 꺼보고, 다시 둘 다 꺼봤다. 결론은 "리퀴드 글래스는 나쁘다"가 아니었다. 기본 블로그 shell이 효과에 기대고 있으면 문제를 분리하기 어렵다는 쪽이었다.

## 기준

첫 번째 기준은 가독성이다. 배경이 흐려지고 빛이 굴절되어도 텍스트 대비는 충분해야 한다. 특히 본문은 유리 뒤에 두지 않는다. 긴 글은 종이처럼 안정적인 표면 위에 있어야 한다.

두 번째 기준은 폴백이다. MDN 문서 기준으로 `backdrop-filter`는 최신 브라우저에서는 넓게 쓸 수 있는 상태가 되었지만, 오래된 기기나 브라우저까지 항상 같게 동작한다고 보면 곤란하다. [`backdrop-filter`](https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/Properties/backdrop-filter)는 뒤쪽 픽셀에 blur, contrast, brightness 같은 필터를 적용하므로 배경과 투명도 조건에도 영향을 받는다. 지원 여부보다 더 현실적인 문제는 "어디에 얼마나 크게 쓰는가"였다.

세 번째 기준은 절제다. 효과가 많아질수록 사이트는 빠르게 피곤해진다. 개인 블로그라면 멋보다 읽는 리듬이 먼저다.

## 위험했던 조합

돌아보면 위험한 조합은 대략 이랬다.

- 전역 Header 전체를 React `client:only` island로 만든다.
- 그 안에 검색, 테마, 굴절 버튼, 상태 관리를 함께 넣는다.
- 넓은 배경에 `backdrop-filter`와 큰 `filter: blur`를 겹친다.
- 전역 `pointermove`에서 매번 DOM 위치를 읽고 CSS 변수를 갱신한다.
- 본문 주변까지 반투명 표면으로 감싸 글자 대비를 흔든다.

각각은 단독으로는 괜찮아 보일 수 있다. 하지만 한꺼번에 들어가면 블로그의 기본 화면이 효과 실험실이 된다. 글을 읽기도 전에 브라우저가 먼저 바빠진다.

그래서 지금의 기준은 반대로 잡았다. Header, footer, 글 카드, 본문 레이아웃은 Astro 정적 HTML로 둔다. React는 검색과 테마처럼 실제 상태가 필요한 곳에만 쓴다. Pagefind도 검색창을 열기 전까지는 불러오지 않는다. 효과가 없어도 블로그가 완성되어 있어야 하고, 효과는 그 위에 얹히는 옵션이어야 한다.

## 어디에 쓰면 좋은가

리퀴드 글래스가 어울리는 곳도 있다. 상단 내비게이션, 검색 다이얼로그 외곽, 작은 액션 버튼, 현재 상태를 보여주는 칩처럼 짧게 보고 지나가는 UI에는 잘 맞는다. 이런 곳에서는 표면의 질감이 사용자의 시선을 안내한다.

반대로 본문, 긴 설명, 코드 블록, 검색 결과 텍스트에는 맞지 않는다. 독자가 오래 바라보는 영역에는 장식보다 대비와 행간이 더 중요하다. 글래스 효과가 "눈에 띄는 장식"에서 "읽기 경험을 돕는 표면"으로 내려오지 못하면 실패다.

CSS도 이렇게 시작하는 편이 낫다.

```css
.glass-control {
  background: color-mix(in srgb, var(--surface) 84%, transparent);
  border: 1px solid color-mix(in srgb, var(--line) 70%, white 18%);
  box-shadow: var(--shadow-soft);
}

@supports (backdrop-filter: blur(12px)) {
  .glass-control[data-fx='enhanced'] {
    backdrop-filter: blur(12px) saturate(115%);
  }
}
```

기본값은 읽히는 표면이고, blur는 지원되는 환경에서만 켜는 강화 옵션이다. 강한 굴절이나 SVG displacement는 더 늦게, 더 작은 영역에서 따로 실험해야 한다.

## 지금의 결론

효과를 버렸다기보다는 순서를 바꿨다. 먼저 정적 shell, 본문 폭, 모바일 줄바꿈, 검색과 태그 탐색을 안정화한다. 그다음 작은 영역에만 시각 효과를 붙인다.

블로그의 시각 효과는 독자가 글을 더 오래 읽게 만들 때만 성공한다. 캡처 이미지에서 멋져 보이는데 본문을 읽기 피곤하다면, 그건 좋은 UI가 아니라 좋은 스크린샷에 가깝다.
